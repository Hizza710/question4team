# 課題名
360° BOARD for TEAM — Let’s Share Unique View & Wish of Ours, Freely.
チームで何かを決める時に、全員の声と希望を即座に拾って、動けるようにするリーダーとチームを支えるツール

## 課題内容（どんな作品か）

(背景)
・チームで何かを決める時に、全員に一人ずつ話を聞くと時間がかかる
・一方で、一人ずつの話を聞いて即座に決めたいこともある
・そんな時に、チームでの合意形成を助けるリーダーとチームを支えるツールに
・一つの問いをいれるだけで、即座に声を集めることができ、誰もが一目でチームの声がわかる
・また全員の投稿内容をテキストでコピペしたり、スクショして保存したりと、活用しやすいように

(内容)
・チームの誰かが問いを入力し、ID番号をいれて、問いを入力して、URLを発行
・そのURLに意見を書くと、全員の声が一目に集まってくる（板チョコのピースが増えていくよ）
・最後にまとめ役の人は結論を書き込むことができ、皆に納得感ある意見を送ることができる
・全員を大切にするリーダー、そして、チーム全員が互いの異なる声を聞く、心強いツールに

## アプリのデプロイURL
　https://hizza710.github.io/question4team/
 
## ⑤工夫した点・こだわった点
・Firebaseの特性を学ぶと、ID管理で誰がいつアクセスしても、見られることだと考え、実装してみた
・メモ帳アプリ作成時は、本人さえわかれば進められるが、今回は誰もが見やすいUIUXにと工夫した
・特にこだわったのは出力は板チョコのように綺麗に並べること。ここで一度すべて決してHTMLでいれるという技を覚えた。・またその出力をコピーして次のアクションに繋げやすいユーザビリティにこだわった。スクショ機能を理解した。
・一度、すぐに使いたくなり、使えるというレベルを目指して仕上げた。デザインより前にまず使えることを目指してみた。

## ⑥難しかった点・次回トライしたいこと（又は機能）
・今はシンプルにCSSでの指示だけでのデザインにしていて、色々変えたいところがある(板チョコらしさの追求など)
・前回いろんな要素をランダムに挙げると重なる難しさがあったため、構造をすっきり整理する方向でつくり見やすくなった
・APIkeyをあげないと動かないため、どのように実際に実装するかを授業で学んでいきたい
・またもう一つ追加したい機能があり、それはAPIが必要になるため、次回それも学び、進化させたい
